# EOTrans - 在线文件传输工具

基于 WebRTC 技术的点对点文件传输工具，无需服务器，端到端加密传输。

## 功能特点

- 📤 **发送文件**：选择本地文件，生成8位随机数字码
- 📥 **接收文件**：输入发送者的8位数字码即可建立连接
- 🔒 **端到端加密**：P2P直连，数据不经过第三方服务器
- 📱 **拖拽上传**：支持拖拽文件到上传区域
- ⚡ **文件限制**：单个文件最大 25MB
- 🌐 **纯前端**：基于 EdgeOnePage，无需后端服务

## 使用方法

### 发送文件

1. 点击"发送文件"按钮
2. 选择要传输的文件（支持点击或拖拽）
3. 系统生成8位数字码，显示在页面上
4. 将这8位数字告诉接收者
5. 等待接收者连接，文件自动传输

### 接收文件

1. 点击"接收文件"按钮
2. 输入发送者提供的8位数字码
3. 点击"开始接收"
4. 等待连接建立，文件自动下载

## 技术栈

- **WebRTC**：RTCPeerConnection + RTCDataChannel
- **信令机制**：基于 LocalStorage 的简单信令交换
- **STUN 服务器**：使用 Google 公共 STUN 服务器
- **前端**：纯 HTML/CSS/JavaScript，无框架依赖

## 部署到 EdgeOnePage

### 方式一：直接上传

1. 将 `index.html` 和 `app.js` 上传到 EdgeOnePage
2. 确保两个文件在同一目录下

### 方式二：使用安装脚本

```bash
# 运行安装脚本
bash install.sh

# 将 dist 目录下的文件上传到 EdgeOnePage
```

### 本地测试

```bash
# 使用 Node.js 启动本地服务器
npx serve .

# 或
npm run dev
```

然后访问 `http://localhost:3000`

## 注意事项

1. **浏览器兼容性**：需要使用支持 WebRTC 的现代浏览器（Chrome, Firefox, Safari, Edge）
2. **HTTPS**：由于 WebRTC 安全限制，建议通过 HTTPS 访问
3. **同一网络测试**：LocalStorage 信令需要在同一浏览器的不同标签页测试，或使用同一局域网内的不同设备
4. **信令有效期**：生成的8位数字码有效期为5分钟

## 工作原理

```
发送者                    信令服务(LocalStorage)                  接收者
  |                                |                                 |
  |---- 1. 生成8位码 ------------->|                                 |
  |---- 2. 创建 Offer ------------>|                                 |
  |                                |<---- 3. 输入8位码 --------------|
  |                                |---- 4. 返回 Offer -------------->|
  |                                |<---- 5. 创建 Answer ------------|
  |<---- 6. 获取 Answer ---------->|                                 |
  |<==============================P2P 连接建立=====================>|
  |<==============================文件传输=========================>|
```
